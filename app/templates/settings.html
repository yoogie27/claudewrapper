GitHub Token Setup Guide

1
Choose token type

For company/org repos, you need either:

  • Fine-grained token (recommended) — scoped to specific repos
  • Classic token — broader access, simpler setup
2
Create the token

Go to github.com/settings/tokens

Fine-grained token:
  1. "Generate new token" → "Fine-grained token"
  2. Resource owner: Select your organization (not personal)
  3. Repository access: "Only select repositories" → pick your repos
  4. Permissions:
    • Contents: Read and write
    • Pull requests: Read and write
    • Metadata: Read-only (auto-selected)
Classic token:
  1. "Generate new token" → "Classic"
  2. Select scope: repo (full control of private repos)
3
Authorize for SSO (org repos only)

If your org uses SAML SSO:

  1. After creating the token, find it in the token list
  2. Click "Configure SSO" next to the token
  3. Click "Authorize" for your organization

This step is the #1 reason tokens fail on org repos. Without SSO authorization, you get 404 errors.

4
Add to .env 
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx

Then restart ClaudeWrapper.

5
Verify

Click "Run Check" above to validate the token, then check each project.
